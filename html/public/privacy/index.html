Poopy Privacy Policy

Last updated: August 18, 2025

General Terms

Poopy only stores the minimum data required for features to function. We do not sell, share, or trade your information. What you do on Discord stays your business.

Poopy processes media and commands on demand but does not log content permanently unless server staff enable saving features. These terms apply only to the official Poopy instance. If you are using a self-hosted copy, please contact the instance’s hoster for details on their policies.

Local & Temporary Processing

Most of Poopy’s processing happens live in Discord’s memory. Data such as command execution, cooldowns, or temporary arrays is deleted once no longer needed.

When using certain features, the following may be stored briefly in memory:

  • User IDs, server IDs, and channel IDs (to know who ran what command).
  • Internal execution info such as cooldowns, promises, and arrays.
  • Temporary command states, which are cleared automatically.
  • Uploaded media (images, videos, GIFs) is cached while transformations are applied, then automatically discarded.
  • Generated media (such as processed images or GIFs) may be cached briefly to deliver results but is not permanently stored.

This data is not permanent and is automatically removed after use.

Saving

If a server’s staff enables saving features, Poopy may store:

  • Saved messages, for optional functionalities like Markov chains or random retrieval.
  • Custom commands, templates, or server settings.
  • Keyword and function settings that allow customized command behavior.
  • Minimal metadata (like counters for messages sent).

By default, message and command storage is disabled. It must be enabled by server staff.

Stored messages are automatically cleared after 30 days unless specifically preserved by server settings. Stored commands, templates, and server settings remain until deleted by administrators.

Encryption

All saved text-based data (messages, commands, templates, and settings) is encrypted with a key stored in environment variables. Temporary runtime data (like cooldowns, exec counts, or media files) is never written to disk and is not encrypted, since it is automatically discarded after processing.

Analytics

Poopy does not use analytics, tracking, or cookies. No identifiable information is collected for statistics.

Web & Hosting

When using Poopy’s media features, files are processed using tools such as FFmpeg and ImageMagick. These files are temporarily cached for processing and delivery, then discarded. Discord’s own servers handle hosting for attachments and webhooks.

User Rights

  • Server owners control whether saving features are enabled.
  • Users may request the removal of their saved messages from server staff.
  • Users may request removal of their custom commands or templates at any time.
  • Generated content is not stored permanently by Poopy. If a server stores generated outputs, removal is the responsibility of that server’s administrators.

Contact

If you have questions or concerns, contact the developer via Discord or the official support server.
